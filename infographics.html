🔧

BooleanEnum Pattern

⚠️ The Problem

Groq's Llama-3.1 hallucinates JSON booleans as strings ("true" vs true), causing Pydantic validation crashes.

✅ The Solution

Engineered custom BooleanEnum(str, Enum) with @field_validator that safely coerces strings, booleans, and integers.

💡 Impact

Demonstrates defensive boundary coding against LLM instability. All downstream code remains type-safe.

🛡️

Graceful RAG Fallback

⚠️ The Problem

FAISS requires 400MB+ embedding models. Failure to download or insufficient RAM crashes standard RAG pipelines silently.

✅ The Solution

HAS_FAISS flag with seamless fallback to Heuristic Keyword Scoring (Title=2pts, Content=1pt, Tags=1pt).

💡 Impact

Absolute fault tolerance. System never fails; knowledge retrieval always returns contextual results.

🔐

Database Abstraction Layer

⚠️ The Problem

Most applications lock into a single database vendor, making scaling or migration painful.

✅ The Solution

Built with SQLAlchemy ORM. Currently SQLite, but any SQL database (PostgreSQL, MySQL, etc.) works without code changes. Only connection string differs.

💡 Impact

Zero-cost database migration. Dev with SQLite, prod with PostgreSQL—no refactoring required.

🎨

Pure REST API Decoupling

⚠️ The Problem

Tightly coupled frontend-backend = monolith. Can't swap backends, scale independently, or deploy separately.

✅ The Solution

Frontend speaks only via HTTP REST calls to 3 API endpoints. Streamlit app is framework-agnostic; backend implementation is irrelevant.

💡 Impact

Frontend deployable to Vercel/Netlify; backend to AWS/GCP/on-prem. Independent scaling.
